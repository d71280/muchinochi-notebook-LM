# サンプルドキュメント - Notebook LM Report Automation

このドキュメントは、Notebook LMレポート生成システムのテスト用サンプルです。

## プロジェクト概要

Claude Code SkillsとGitHub Actionsを組み合わせて、Notebook LMによるレポート作成を自動化するシステムを構築しています。

### 主な機能

1. **自動ソース収集**: `sources/` ディレクトリ内のドキュメントを自動的に収集
2. **Notebook LM統合**: Google Cloud APIまたはサードパーティAPIを使用
3. **マルチフォーマット出力**: Markdown、JSON、HTML形式でレポートを生成
4. **CI/CD統合**: GitHub Actionsで完全自動化

## 技術スタック

- **Claude Code**: AI支援による自動化
- **GitHub Actions**: CI/CDパイプライン
- **Python**: バックエンド処理
- **Notebook LM**: レポート生成エンジン
- **Google Cloud**: APIインフラストラクチャ

## ユースケース

### 1. 週次レポート生成

毎週月曜日に自動的に以下を実行：
- 前週の活動ログを収集
- 重要なトピックを分析
- サマリーレポートを生成

### 2. プルリクエストレビュー

PRが作成されると：
- 関連ドキュメントを分析
- 変更点の影響を評価
- レビューレポートを作成

### 3. ドキュメント統合

複数のドキュメントから：
- 統合された知識ベースを構築
- クロスリファレンスを生成
- Q&Aセクションを自動生成
